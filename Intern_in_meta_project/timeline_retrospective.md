### ✅ Week 1: 회고 정리
#### 📅 7월 2일: 첫 스프린트 시작
- 영향: 프로젝트의 초석을 다지며 팀의 초점과 우선순위 설정.

- 통찰: 스프린트의 시작이 팀 동기부여에 중요한 역할을 함을 인식.

- 액션 플랜: 향후 스프린트 시작 전 명확한 목표와 역할 분담을 통해 팀의 몰입도 향상시키기.

#### 📅 7월 3일: 고객사 미팅
- 영향: 고객 요구사항을 직접 청취하며 프로젝트 방향성 구체화.

- 통찰: 고객의 니즈를 정확히 파악하는 것이 개발의 핵심임을 체감.

- 액션 플랜: 고객 피드백을 정기적으로 수집하고 요구사항 문서화 프로세스 강화하기.

#### 📅 7월 4일: 상품 목록 페이지 요구사항 분석서 작성
- 영향: 기능 구현의 기준점 마련 및 팀 내 작업 방향 통일.

- 통찰: 분석서 작성이 개발 효율성과 커뮤니케이션에 큰 영향을 미침.

- 액션 플랜: 사용자 시나리오 기반으로 요구사항 분석서를 작성하여 실사용자 중심 개발 유도.

#### 📅 7월 5일: Git/GitHub 사용법 익히기
- 영향: 협업을 위한 버전 관리 체계 확립.

- 통찰: Git은 단순한 도구가 아닌 협업의 핵심 인프라임을 인식.

- 액션 플랜: 브랜치 전략과 커밋 메시지 규칙을 팀 내 표준으로 정립하기.

#### 📅 7월 6일: 프로젝트 회고
- 영향: 첫 주의 성과와 문제점을 되짚으며 개선 방향 도출.

- 통찰: 회고는 단순 리뷰가 아닌 팀 성장의 기회임을 인식.

- 액션 플랜: 매주 회고를 정례화하고, 액션 아이템을 다음 스프린트에 반영하기.

### ✅ Week 2: 회고 정리
#### 📅 7월 8일: Jest 활용 단위 테스트
- 영향: 컴포넌트 기능을 사전에 검증함으로써 개발 안정성과 신뢰도 향상.

- 통찰: 테스트는 단순한 확인이 아니라, 코드 품질을 보장하는 핵심 도구임을 인식.

- 액션 플랜: 주요 기능마다 테스트 케이스를 작성하고, 테스트 자동화를 위한 환경을 지속적으로 개선하기.

#### 📅 7월 9일: 상품 목록 페이지 문서화
- 영향: 기능 흐름과 요구사항을 명확히 정리하여 팀 내 커뮤니케이션 효율성 증대.

- 통찰: 문서화는 개발자뿐 아니라 디자이너, 기획자와의 협업에도 중요한 연결 고리임을 인식.

- 액션 플랜: 기능별 문서 템플릿을 정립하고, 변경사항 발생 시 문서 갱신 프로세스를 자동화하기.

#### 📅 7월 10일: GitHub Pages 및 Netlify 배포
- 영향: 초기 결과물을 외부에 공유함으로써 피드백 수집 및 개선 방향 도출 가능.

- 통찰: 배포는 단순한 기술 작업이 아니라, 사용자와의 첫 접점이라는 점에서 전략적으로 접근해야 함을 인식.

- 액션 플랜: 배포 자동화(CI/CD) 파이프라인을 구축하고, staging 환경과 production 환경을 분리하여 안정성 확보하기.

- 영향: 장바구니, 결제, 상품 목록 등 개별 기능을 하나의 흐름으로 연결하여 사용자 경험 완성도 향상.

- 통찰: 기능 구현보다 중요한 것은 기능 간의 연결성과 흐름이라는 점을 실감.

- 액션 플랜: 페이지 간 데이터 흐름을 명확히 하고, 사용자 여정(User Journey)을 기반으로 기능 배치 최적화하기.

#### 📅 8월 13일: React Router 활용
- 영향: 페이지 간 이동을 자연스럽게 처리하여 SPA(Single Page Application)의 장점을 극대화.

- 통찰: 라우팅은 단순한 URL 연결이 아니라, 사용자 맥락(Context)을 고려한 설계가 필요함을 인식.

- 액션 플랜: 라우터 구조를 문서화하고, 동적 라우팅 및 보호된 라우트(Protected Route) 등 고급 기능도 점진적으로 도입하기.

### ✅ Week 8: 회고 정리
#### 📅 8월 18일: Cypress로 핵심 기능 테스트
- 영향: 사용자 관점에서의 시나리오 테스트를 통해 기능 안정성 확보.

- 통찰: E2E 테스트는 단순한 자동화가 아니라, 실제 사용자 행동을 시뮬레이션하는 중요한 품질 보증 수단임을 인식.

- 액션 플랜: 주요 사용자 흐름(로그인, 장바구니, 결제 등)에 대한 Cypress 테스트 시나리오를 정리하고, 배포 전 자동 실행되도록 설정하기.

#### 📅 8월 19일: 상세 페이지 구현
- 영향: 상품별 정보 제공으로 사용자 만족도 및 구매 전환율 향상.

- 통찰: 상세 페이지는 단순한 정보 나열이 아니라, 사용자 설득을 위한 전략적 UI 설계가 필요함을 인식.

- 액션 플랜: 상품 정보 구성 요소(이미지, 설명, 리뷰 등)를 컴포넌트화하고, 향후 확장 가능한 구조로 설계하기.

#### 📅 8월 20일: 전체 코드 리팩터링
- 영향: 코드 일관성과 유지보수성 향상, 성능 개선.

- 통찰: 리팩터링은 기능 추가보다 더 큰 기술적 가치를 지니며, 장기적인 프로젝트 안정성에 기여함을 인식.

- 액션 플랜: 반복되는 로직을 공통 모듈로 분리하고, ESLint 및 Prettier 설정을 통해 코드 스타일 통일하기.

#### 📅 8월 21일: 최종 완성한 쇼핑몰 페이지 문서화
- 영향: 프로젝트 결과물을 명확히 정리하여 외부 공유 및 유지보수 기반 마련.

- 통찰: 문서화는 프로젝트의 마무리가 아니라, 다음 단계로 연결되는 다리임을 인식.

- 액션 플랜: 기능별 문서, API 명세, 사용자 흐름 등을 포함한 종합 문서 패키지를 구성하고, README에 요약 정리하기.
